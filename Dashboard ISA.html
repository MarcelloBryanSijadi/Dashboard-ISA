Ringkasan Audit & Strategi Keamanan

📄 Latar Belakang
Di era transformasi digital, tata kelola teknologi informasi (TI) menjadi pilar utama efektivitas layanan publik. Dukcapil Minahasa Utara mengoperasikan Sistem Informasi Administrasi Kependudukan (SIAK) yang terpusat dan aplikasi mandiri SIGAP untuk layanan online. Namun, ditemukan risiko kritis berupa ketergantungan tinggi pada jaringan internet, human error dalam penginputan data, dan pembatasan hak akses yang menyebabkan ketergantungan pada operator tertentu. Audit menggunakan framework ITIL V4 dilakukan untuk mengevaluasi dan memitigasi risiko-risiko tersebut.
🎯 Tujuan Penelitian
  1. Mengevaluasi keamanan data pribadi penduduk dari akses tidak sah menggunakan framework ITIL V4.
  2. Menilai tingkat ketersediaan (availability) infrastruktur jaringan dalam mendukung pelayanan.
  3. Menjamin akurasi informasi kependudukan melalui penguatan kontrol validasi sistem.
  4. Mengukur efisiensi sistem SIAK dan SIGAP dalam mempercepat proses pelayanan publik.

✅ Kekuatan
SIAK mendukung seluruh 20 layanan utama kependudukan (KK, KTP, Akta)

Tidak ada SOP formal dan incident log untuk penanganan gangguan

Human error input data: jenis kelamin tertukar, tanggal lahir typo
Monitoring tidak real-time, tidak ada notifikasi otomatis
Tidak ada kebijakan keamanan informasi yang terdokumentasi formal

Rekomendasi & Mitigasi Strategis

R1 — CRITICAL
Gangguan Jaringan Internet
Rekomendasi
Multi ISP redundancy + monitoring real-time + SOP penanganan gangguan jaringan
R2 — CRITICAL
Human Error Input Data
Rekomendasi
Validasi otomatis (dropdown, auto-fill) + pelatihan operator berkala + audit kualitas data
R5 — CRITICAL
Penyalahgunaan Hak Akses
Rekomendasi
Implementasi Role-Based Access Control (RBAC) + audit akses berkala + logging aktivitas
R4 — HIGH
Ketergantungan Sistem Pusat
Rekomendasi
Koordinasi dengan pusat + fallback manual + prosedur eskalasi gangguan sistem nasional
R6 — HIGH
Monitoring Tidak Real-Time
Rekomendasi
Implementasi sistem monitoring terintegrasi + alert system otomatis + dashboard monitoring
R7 — HIGH
Tidak Ada SOP Layanan TI
Rekomendasi
Penyusunan SOP seluruh proses layanan TI + penetapan SLA sebagai standar waktu penyelesaian
R10 — HIGH
Ketergantungan Infrastruktur Nasional
Rekomendasi
SLA dengan penyedia layanan pusat + prosedur fallback operasional daerah
R3 — MODERATE
Keterbatasan Aplikasi SIGAP
Rekomendasi
Percepatan pengembangan SIGAP + integrasi penuh dengan SIAK + perluasan cakupan layanan

🎤 Highlight Wawancara — Bapak Jolly Tuwaidan (Narasumber)
Sistem yang Digunakan
"SIAK dari kantor pusat, sudah dari 2008. SIGAP dari Minahasa Utara sendiri — aplikasi Android, baru akhir tahun lalu, masih tahap pengembangan/uji coba."
Hak Akses
"Ada SK Kepala Dinas. Tidak semua pegawai bisa akses karena menyangkut data pribadi. Ada tingkatan level — ada yang hanya bisa Dukcapil, ada yang bisa semua."
Gangguan & Backup
"Pernah terjadi gangguan jaringan — dulu hanya 1 Telkom Fiber, sekarang sudah 2 jaringan berbeda. Backup database di pusat, di sini hanya agar data tidak bocor."
Human Error & Evaluasi
"Human error pasti ada — biasanya jenis kelamin tertukar, tanggal lahir typo. Evaluasi ada, tapi karena sistemnya nasional, evaluasinya saat Bintek dari pusat."
